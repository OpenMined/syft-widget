Why We Built This

widgets should be resilient

🚫 Widgets break

Traditional widgets fail when servers go down or networks disconnect. Your data visualization shouldn't depend on infrastructure.

🏢 Updates are hard

Getting live updates from filesystems into Jupyter notebooks requires complex setup. It should be one line of code.

⛓️ Deployment is painful

Moving from notebook prototype to production deployment involves rewriting everything. The same widget should work everywhere.

Under the Hood

Three-mode architecture for ultimate resilience

# Mode 1: SyftBox (Preferred)
widget = MyWidget()  # Automatically connects to SyftBox infrastructure
# ✅ Live filesystem monitoring
# ✅ Multi-client support
# ✅ Auto-deployment from GitHub

# Mode 2: Thread (Automatic Fallback)
widget = MyWidget(mode="thread")  # Python thread monitors files
# ✅ Works without infrastructure
# ✅ Seamless handoff to SyftBox
# ✅ Local file watching

# Mode 3: Checkpoint (Last Resort)
widget = MyWidget(mode="checkpoint")  # Shows last saved state
# ✅ Never shows errors
# ✅ Manual refresh available
# ✅ Cached data persistence

The Complete API

build widgets and APIs with minimal code

Create a Widget

from syft_widget import APIDisplay

class BirdWidget(APIDisplay):
    def __init__(self):
        super().__init__(endpoints=["/api/birds"])
    
    def render_content(self, data, server_type):
        birds = data.get("/api/birds", {}).get("count", 0)
        return f"
🦜 {birds} birds spotted
"

Add REST APIs

from syft_widget import register_endpoint

@register_endpoint("/api/birds")
def get_birds(request):
    return {"birds": load_bird_data()}

@register_endpoint("/api/add", methods=["POST"])
def add_bird(request):
    save_bird(request.json)
    return {"success": True}
