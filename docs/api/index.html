Display Classes

APIDisplay(mode=None)

Base class for creating resilient widgets with automatic fallback between SyftBox, thread, and checkpoint modes. Inherit from this class to build widgets that never break.

Parameters
mode str, optional Force a specific mode: 'syftbox', 'thread', or 'checkpoint'. If None, tries modes in order.
Methods to Override
render() Must return HTML string to display. Called on updates.
Example
from syft_widget import APIDisplay

class MyWidget(APIDisplay):
    def render(self):
        # This method is called automatically on updates
        return f"
Widget content at {datetime.now()}
"

# Widget automatically tries SyftBox → Thread → Checkpoint
widget = MyWidget()
widget  # Display in Jupyter
widget.get_current_mode()

Get the current connection mode of the widget. Useful for debugging and status displays.

Returns
str Current mode: 'syftbox', 'thread', or 'checkpoint'
Example
widget = MyWidget()
mode = widget.get_current_mode()
print(f"Running in {mode} mode")

# Use in render method for status
def render(self):
    mode = self.get_current_mode()
    return f"
Mode: {mode}
"

Endpoint Decorators

@register_endpoint(path, methods=["GET"])

Decorator to register a function as a REST API endpoint. The endpoint is automatically available when running in SyftBox mode with CORS headers handled automatically.

Parameters
path str URL path for the endpoint (e.g., "/api/data")
methods list[str] HTTP methods to accept. Default: ["GET"]
Function Requirements
request Function must accept a FastAPI Request object
return Must return JSON-serializable data
Example
from syft_widget import register_endpoint

@register_endpoint("/api/birds")
def get_birds(request):
    # Automatic CORS handling
    return {"birds": ["Blue Jay", "Cardinal", "Robin"]}

@register_endpoint("/api/add_bird", methods=["POST"])
def add_bird(request):
    data = request.json  # Get POST data
    save_bird(data)
    return {"success": True, "bird": data["name"]}
get_all_endpoints()

Get a list of all registered endpoints in the current package. Useful for debugging and API documentation generation.

Returns
list[dict] List of endpoint info with path, methods, and function details
Example
from syft_widget import get_all_endpoints

# List all registered endpoints
endpoints = get_all_endpoints()
for ep in endpoints:
    print(f"{ep['methods']} {ep['path']} → {ep['function'].__name__}")

Package Utilities

create_package(name, github_username, **kwargs)

Create a complete syft-widget package structure with all necessary files, ready for development and deployment to GitHub/PyPI.

Parameters
name str Package name (e.g., "my-widget")
github_username str Your GitHub username for repository URL
display_name str, optional Human-readable name for the widget
description str, optional Package description
Returns
Path Path to the created package directory
Example
from syft_widget import create_package

# Create a new widget package
package_dir = create_package(
    "bird-watcher",
    "myusername",
    display_name="Bird Watcher Widget",
    description="Live bird sighting tracker"
)

print(f"Package created at: {package_dir}")
# Creates:
# bird-watcher/
#   ├── displays/
#   ├── endpoints/
#   ├── pyproject.toml
#   └── ...

Infrastructure Management

start_infrastructure(port=8000)

Start the syft-widget development server for testing endpoints and widgets locally. This is primarily for development - production deployment happens via SyftBox.

Parameters
port int Port to run the server on. Default: 8000
Example
from syft_widget import start_infrastructure

# Start development server
start_infrastructure(port=8080)
# Server running at http://localhost:8080
# Your endpoints available at http://localhost:8080/api/*
stop_infrastructure()

Stop the running development server and clean up resources.

Example
from syft_widget import stop_infrastructure

# Stop the server
stop_infrastructure()
